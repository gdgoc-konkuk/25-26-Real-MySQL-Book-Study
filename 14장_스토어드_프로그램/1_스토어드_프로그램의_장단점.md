## 14.1.1 스토어드 프로그램의 장점

1. 데이터베이스의 보안 향상
   - 자체적인 보안 설정 기능 보유
   - 스토어드 프로그램 단위로 실행 권한 부여 가능
   - 특정 테이블의 읽기와 쓰기, 또는 특정 칼럼에 대해서만 권한을 설정하는 등 세밀한 권한 제어가 가능

2. 기능의 추상화
   - 특정한 기능 (ex) 일련번호 발급)들을 스토어드 프로그램으로 작성해서 등록해두면 개발 언어나 도구와 관계 없이, 해당 기능을 활용할 수 있다.

3. 네트워크 소요 시간 절감
   - 하나의 프로그램에서 100번, 200번 처럼 굉장히 많이 실행되어야 하는 쿼리를 스토어드 프로그램으로 구현해둔다면, 스토어드 프로그램을 호출할 때 한 번만 네트워크를 경유하면 되도록 최적화할 수 있어 성능 개선이 가능하다.
  
4. 절차적 기능 구현
   - SQL 쿼리에서는 복잡한 형태의 IF나 WHILE 과 같은 제어 문장을 사용할 수 없다. 그데 반해 스토어드 프로그램은 DBMS 서버에서 절차적인 기능을 실행할 수 있는 제어 기능을 제공한다.

5. 개발 업무의 구분
   - 순수하게 애플리케이션 개발 조직과 SQL 개발 조직이 구분돼 있는 회사의 경우, DBMS 코드를 개발하는 조직에서는 트랜잭션 단위로 데이터베이스 관련 처리를 하는 스토어드 프로그램을 만들어 API 처럼 제공하고, 애플리케이션 개발자는 스토어드 프로그램을 호출해서 사용하는 형태로 역할을 구분하여 개발 진행이 가능하다.

## 14.1.2 스토어드 프로그램의 단점

1. 낮은 처리 성능
   - 스토어드 프로그램의 처리 성능이 다른 프로그램 언어에 비해 상대적으로 떨어진다.

2. 애플리케이션 코드의 조각화 
   - 각 기능을 담당하는 프로그램 코드가 자바와 MySQL 스토어드 프로그램으로 분산된다면 애플리케이션의 설치나 배포가 더 복잡해지고 유지보수 또한 어려워질 수 있다.
