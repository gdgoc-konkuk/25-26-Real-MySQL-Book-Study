## 1. 개요
데이터베이스의 확장성(Scalability)과 가용성(Availability)을 위해 데이터베이스 서버의 확장이 필수적이다.
복제란 한 서버에서 다른 서버로 데이터가 동기화되는 것을 말하며, 원본 데이터를 가진 서버를 소스(Source) 서버, 복제된 데이터를 가지는 서버를 레플리카(Replica) 서버라고 부른다.

복제를 통해 얻을 수 있는 대표적인 이점

1. 스케일 아웃(Scale-out)

스케일 업(Scale-up): 한 대의 DB 서버에 대해 사양을 업그레이드하여 늘어난 트래픽을 처리하는 방식
스케일 아웃(Scale-out): 동일한 데이터를 가진 DB 서버를 한 대 이상 더 만들어 늘어난 트래픽을 처리할 수 있도록 하는 방식

2. 데이터 백업

사용자의 실수로 데이터가 삭제되는 경우를 대비하기 위해 데이터 백업이 필수적이다. 동일한 DB 서버 내에서 백업 프로그램과 DBMS가 서버의 자원을 공유하는 경우 쿼리의 속도가 느려져 서비스에 문제가 발생할 수 있다. 이 같은 문제를 방지하기 위해 복제를 사용해 레플리카 서버를 구축하고, 데이터 백업은 레플리카 서버에서 실행한다. 이렇게 구축된 백업용 레플리카 서버는 소스 서버가 문제 생겼을 때를 대비한 대체 서버의 역할을 하기도 한다.

3. 데이터 분석

DB 서버에서는 서비스에서 사용되는 쿼리들 외에 차세대 비즈니스 모델을 발굴하기 위해서 등 분석용 쿼리들을 실행하기도 한다. 이러한 분석용 쿼리는 대량의 데이터를 조회하거나 집계 연산을 하는 등 쿼리 자체가 무거운 경우가 많기 때문에 복제를 사용해 레플리카 서버를 구축해 분석용 쿼리만 전용으로 실행되는 환경을 만드는 것이 좋다.

4. 데이터의 지리적 분산
DB 서버와 애플리케이션 서버가 서로 지리적으로 떨어져 있는 경우 두 서버 간의 통신 시간 및 속도에 영향을 받는다. 복제를 사용해 애플리케이션 서버가 위치한 곳에 레플리카 서버를 새로 구축해 응답 속도를 개선할 수 있다.
